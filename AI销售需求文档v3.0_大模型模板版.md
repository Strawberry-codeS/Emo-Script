# 一、背景

## 1.1 业务背景

瑞思教育采用直营+加盟模式，覆盖全国多个城市，各地校区由地推+TMK+CC顾问承担从线索跟进到现场签单的完整销售链路。销售转化率是校区最核心的经营指标。

### 行业现状

少儿英语培训行业在2026年面临以下挑战：
- **线索成本持续上升**：地推CVC 85-100元/条，线上投放成本逐年增长
- **竞品竞争激烈**：VIPKID、51Talk、伴鱼少儿英语、英孚少儿英语等竞品持续降价、优化服务
- **人员流动性高**：优秀顾问流失导致校区业绩波动，新人培养周期长（3-6个月）
- **家长决策周期延长**：从首次接触到报名平均需要2-4周，对比多家机构成为常态

### 当前销售体系三大根本性问题

| 问题 | 具体表现 | 业务影响 |
|------|---------|---------|
| 经验无法沉淀 | 优秀顾问的话术与方法存于个人，人员流动后知识流失，新人成长靠自己摸索 | 新人达标周期长，整体转化率低 |
| 过程不可管 | 管理层依赖人工抽检和手工统计，每天重复操作3次以上，销售过程是黑盒 | 无法及时发现和干预问题，管理效率低 |
| 转化不稳定 | 业绩高度依赖少数能力强的顾问，名将一走业绩即塌 | 校区经营风险高，业绩波动大 |

### 转化漏斗基准值

| 环节 | 指标 | 目标值 | 上海现状 |
|------|------|--------|---------|
| 获客→下发 | 下发率 | ≥35% | 待确认 |
| 下发→电转 | 电转率 | ≥30% | 约24% |
| 电转→到访 | 到访率 | ≥80% | 约70-79% |
| 到访→面转 | 面转率 | ≥40% | 约31%（部分校区15%） |
| 总转率 | 咨询量→关单 | ≥8% | 7.5% |

### 业务价值

- **提升转化效率**：通过AI赋能，目标将面转率从31%提升至40%以上
- **降低人力成本**：AI外呼筛选低意向线索，TMK人力集中在高价值线索上
- **标准化销售流程**：优秀话术系统化沉淀，新人达标周期缩短50%
- **非工作时间零流失**：AI招生顾问7×24h接待，目标非工作时间留资率≥18%

## 1.2 为什么用大模型解决

### 传统方案的局限

| 场景 | 传统方案 | 局限 |
|------|---------|------|
| 通话纪要 | 关键词提取+模板填充 | 无法理解上下文，纪要质量差，不含三性评估 |
| 客户分类 | 规则引擎（关键词匹配） | 覆盖场景有限，无法处理复杂语义（如"你们和英孚比怎么样"） |
| 成长报告 | 顾问手写/套用模板 | 品质极不统一，耗时30分钟以上/份 |
| 话术推荐 | 固定FAQ检索 | 无法根据具体对话上下文动态推荐，体验差 |

### 大模型在本场景的核心优势

1. **语义理解能力**：准确识别家长意图、情绪、客户类型，支持多维度特征工程
2. **自然语言生成**：自动生成≤200字结构化纪要、成长报告、追单话术
3. **上下文感知**：基于完整对话历史动态推荐话术，而非固定FAQ
4. **多模态能力**：支持语音口述→文字报告、Demo视频智能剪辑

### ROI对比

| 方案 | 开发成本 | 效果 | 维护成本 |
|------|---------|------|---------|
| 规则引擎 | 低 | 覆盖率<40%，准确率低 | 高（需持续维护规则） |
| 大模型方案 | 中 | 覆盖率>85%，准确率高 | 中（Prompt调优+评测集维护） |

### 不可替代性

本场景中，客户意图识别、情绪分析、话术动态推荐、成长报告生成等核心功能强依赖自然语言理解与生成能力，传统规则引擎无法胜任。大模型是本场景的最优解。

## 1.3 竞品分析

| 竞品 | AI销售赋能现状 | 测评工具 | 成长报告 | 差异化机会 |
|------|--------------|---------|---------|-----------|
| 英孚少儿英语 | AI外呼+智能客服，但无销售链路AI赋能 | 标准化线上测评，但解读依赖顾问 | 简单学习建议，无个性化报告 | 深度AI销售赋能+个性化成长报告 |
| VIPKID | AI智能排课+学习报告，侧重课后服务 | 线上入学测评，自动分级 | 学习进度报告，但偏数据化 | 线下OMO体验+面资AI赋能 |
| 伴鱼少儿英语 | AI绘本阅读推荐，侧重内容 | 绘本阅读能力测评 | 阅读能力成长报告 | 全链路销售AI赋能，覆盖售前售中售后 |
| 51Talk | AI口语测评+智能纠音 | AI口语水平测试 | 学习效果对比报告 | 线下场景+面资AI辅助 |
| 金宝贝 | 无AI销售工具 | 儿童发展测评（非英语） | "体检报告"式成长报告，专业感强 | 借鉴"体检报告"形式，做英语能力专属报告 |

### 核心差异化优势

1. **全链路AI赋能**：从线索获取到签单追单，每个关键节点都有AI工具支撑
2. **结构化纪要+客户分类**：自动生成≤200字简洁纪要，含三性评估、客户类型、下一步建议
3. **AI成长报告**：参考金宝贝"体检报告"形式，生成专业、个性化的英语能力评估报告
4. **面资Copilot**：实时话术推送、竞品应对、异议处理案例库，降低新人能力门槛
5. **测评表数字化+自动解读**：老师线上勾选，系统自动生成解读说明，傻瓜版设计

## 1.4 产品目标

### 产品定位

> "线索来了，每条都要有结果。让最好的销售方法，复制给每一个人。"

Rise Advisor 由两个协同模块构成：

1. **AI招生顾问（线上侧）**：小程序内智能顾问，7×24h承接家长咨询，自动识别需求，精准推荐课程，引导完成试听预约
2. **AI销售赋能（线下侧）**：覆盖线下销售全链路，从TMK外呼到面资压单，每个关键节点提供AI工具支撑，赋能顾问，提升整体转化水平

### 产品目标

| 目标维度 | 具体目标 | 衡量指标 | 目标值 |
|---------|---------|---------|--------|
| 线上线索接待 | 7×24h全天候，线上线索零流失 | 非工作时间线索留资率 | ≥18% |
| 到访率提升 | 邀约到访转化率提升 | 预约到访率 | ≥35% |
| 面资转化 | 面资工具标准化，降低顾问能力依赖 | 面转率 | ≥40% |
| 经验复制 | 优秀话术系统化，新人快速达标 | 新人达标周期 | 缩短50% |
| 过程可管 | 全链路销售过程数据可见 | 管理层数据看板覆盖率 | 100% |

## 2.2 需求分类

### 功能需求

- **语音理解与生成能力**：通话录音ASR转写、语音口述转文字报告
- **意图识别能力**：家长咨询意图分类、客户六类型分类、情绪四维分析
- **智能推荐能力**：课程推荐、话术推荐、竞品应对推荐
- **内容生成能力**：结构化纪要生成、成长报告生成、追单话术生成
- **多领域应用能力**：覆盖TMK外呼、邀约到访、Demo体验、面资咨询、追单跟进全链路

### 性能需求

| 维度 | 要求 |
|------|------|
| 转写延迟 | 通话转写延迟≤3秒 |
| 报告生成 | 成长报告生成延迟≤30秒 |
| 纪要生成 | 通话结束后3分钟内自动生成 |
| 系统可用性 | ≥99.5% |
| 并发支持 | 支持全国校区同时使用，峰值QPS≥500 |

### 安全需求

- **数据安全**：家长数据加密存储，录音数据脱敏处理
- **隐私保护**：面资录音需家长授权，遵循《个人信息保护法》
- **访问控制**：基于角色的权限控制（CC/校长/区域管理）
- **数据不出境**：所有数据存储和处理在国内完成

### 数据需求

- **埋点数据**：功能使用链路（AI纪要使用率、报告生成次数、Copilot调用次数）
- **正负case数据**：
  - 正case：顾问采纳率高的推荐话术、家长高满意度的报告
  - 负case：顾问修改率高的纪要、家长投诉或不认可的报告
- **业务数据**：面转率、电转率、留资率、新人达标率（用于验收功能价值）

# 五、模型选型

## 5.1 选型约束条件

| 约束条件 | 说明 |
|---------|------|
| 成本约束 | 日均处理通话量约5000通，需控制API调用成本在可接受范围 |
| 延迟要求 | 实时场景（面资Copilot）延迟≤3秒，离线场景（纪要生成）≤3分钟 |
| 数据安全 | 所有数据国内处理，不可出境 |
| 效果要求 | 客户分类准确率≥80%，纪要生成满意度≥75% |

## 5.2 可选模型对比

| 模型 | 适用场景 | 成本 | 延迟 | 优势 | 劣势 |
|------|---------|------|------|------|------|
| 通义千问-Max | 纪要生成、报告生成 | 中 | 中 | 中文理解强，教育领域效果好 | 成本较高 |
| 智谱GLM-4 | 客户分类、意图识别 | 中低 | 低 | 推理能力强，性价比好 | 长文本支持有限 |
| 百川大模型 | 情绪分析、话术推荐 | 低 | 低 | 成本低，适合批量场景 | 复杂任务效果一般 |
| 本地部署Qwen-72B | 全场景 | 高（一次性） | 中 | 数据完全可控，无限调用 | 需GPU资源，运维复杂 |

## 5.3 选型结论

| 节点 | 主模型 | 备用模型 | 理由 |
|------|--------|---------|------|
| ASR转写 | 科大讯飞/阿里云ASR | 字节火山引擎ASR | 中文识别准确率最高 |
| 客户分类/意图识别 | 智谱GLM-4 | 通义千问-Plus | 性价比高，分类准确率高 |
| 纪要生成/报告生成 | 通义千问-Max | 智谱GLM-4 | 长文本生成质量好 |
| 话术推荐/实时提词 | 本地部署Qwen-72B | 智谱GLM-4 | 低延迟要求，需快速响应 |

# 六、Prompt工程设计

## 6.1 System Prompt设计

### 场景一：AI结构化纪要生成

**定义**：模型扮演资深销售分析师，基于通话转写文本，生成结构化销售纪要

**核心指令**：
1. 分析通话内容，提取客户三性评估（意愿性/时间性/经济性）
2. 识别客户类型（竞品对比型/服务敏感型/距离敏感型/价格敏感型/专业敏感型/升学导向型）
3. 提炼核心信息（孩子情况+家长核心诉求）
4. 生成下一步具体可执行的跟进建议
5. 输出字数控制在200字以内

**输出约束**：
- 格式：JSON结构，包含三性、客户类型、核心信息、下一步动作
- 语言：简洁专业，避免套话
- 禁止：不得虚构通话中未提及的信息

**System Prompt示例**：

```
你是一位资深少儿英语销售分析师，擅长从通话中快速提炼关键信息。

你的任务：
1. 评估客户三性：
   - 意愿性：高/中/低
   - 时间性：具体时间或时间段
   - 经济性：有预算/有顾虑/未提及

2. 判断客户类型（可多选）：
   - 竞品对比型：提到其他机构，进行对比
   - 服务敏感型：关注师资、排课、课后服务
   - 距离敏感型：关注校区位置、交通
   - 价格敏感型：频繁询问价格、优惠
   - 专业敏感型：关注教学体系、方法、效果
   - 升学导向型：提及升学时间、考级

3. 核心信息：孩子年龄+基础+家长核心诉求（一句话）

4. 下一步动作：具体可执行的跟进建议（时间+内容）

输出格式：
{"三性": "...", "客户类型": "...", "核心信息": "...", "下一步动作": "..."}

要求：
- 整体字数不超过200字
- 基于通话事实，不虚构信息
- 语言简洁，直接可用
```

### 场景二：AI成长报告生成

**定义**：模型扮演少儿英语教育专家，基于顾问口述和测评数据，生成专业成长报告

**核心指令**：
1. 整合顾问口述内容、测评表数据、Demo课观察
2. 生成六大模块报告：能力现状、课堂观察、同龄参照、培养路径、家校建议、下次跟进
3. 使用专业但家长能理解的语言
4. 自然植入瑞思课程产品

**输出约束**：
- 格式：六大模块结构
- 语气：专业温暖，避免过度营销感
- 禁止：不得编造孩子不存在的能力或问题

## 6.2 Prompt策略

- **Few-shot提示**：在纪要生成和报告生成中，提供2-3个优秀示例作为参考
- **角色设定**：根据不同场景设定不同专家角色（销售分析师、教育专家）
- **输出约束**：使用JSON Schema或明确格式要求，确保输出结构一致
- **安全约束**：添加"不得虚构信息"指令，减少幻觉

## 6.3 Prompt版本管理

# 八、评测体系

## 8.1 评测集构建方法论

### 数据来源

- **历史通话录音**：从COS系统抽取过去3个月的通话录音（覆盖不同客户类型、不同顾问水平）
- **历史成长报告**：收集优秀顾问撰写的报告作为正例
- **真实业务场景**：上海区域试点期间收集的case

### 规模标准

| 场景 | 最小规模 | 目标规模 |
|------|---------|---------|
| 客户分类 | ≥200条 | 500条 |
| 纪要生成 | ≥100条 | 300条 |
| 报告生成 | ≥50条 | 100条 |
| 意图识别 | ≥300条 | 800条 |

### 覆盖度设计

按场景分层构建评测集：
- 客户类型分布：竞品对比型20%、服务敏感型15%、距离敏感型15%、价格敏感型20%、专业敏感型15%、升学导向型15%
- 通话时长分布：短通话(<3min)30%、中通话(3-10min)50%、长通话(>10min)20%

## 8.2 评测维度与评分标准

| 维度 | 权重 | 评分标准（1-5分） |
|------|------|------------------|
| 准确性 | 30% | 5分：完全准确，可直接使用；3分：基本准确，需微调；1分：有明显错误 |
| 完整性 | 25% | 5分：所有要素齐全；3分：缺少次要要素；1分：缺少核心要素 |
| 格式规范 | 15% | 5分：格式完全符合要求；3分：格式基本正确；1分：格式错误 |
| 安全合规 | 20% | 一票否决项：出现虚构信息或不当内容直接判定为不通过 |
| 表达质量 | 10% | 5分：语言流畅专业；3分：表达基本清晰；1分：表达混乱 |

**总分 = 准确性×30% + 完整性×25% + 格式规范×15% + 安全合规×20% + 表达质量×10%**

**安全合规为一票否决项**：该维度≤2分，无论总分多少，判定为不通过

## 8.3 评测集示例

### 客户分类评测示例

| 输入（通话片段） | 预期输出 | 评测标准 |
|----------------|---------|---------|
| "我们孩子之前在英孚学了一年，现在想换，你们和英孚比有什么不同？" | 竞品对比型（置信度≥80%） | 分类正确+置信度合理 |
| "学费多少钱啊？别家都比你们便宜" | 价格敏感型（置信度≥80%） | 分类正确+置信度合理 |

## 8.4 评测执行流程

1. **准备阶段**：构建评测集，标注预期输出
2. **执行阶段**：运行模型，生成结果
3. **评分阶段**：按评分标准打分
4. **分析阶段**：分析Bad Case，归因失败原因
5. **优化阶段**：调整Prompt或模型，重新评测

## 8.5 评测集持续维护

- **Bad Case回流**：线上发现的Bad Case每周回流到评测集，标注场景层级和失败原因
- **定期清理**：每季度审查评测集，剔除过时用例
- **场景扩展同步**：每次新增功能场景时，同步扩充对应的评测
- **版本管理**：评测集使用版本号管理（如eval_v1.0、eval_v1.1）

### 评测触发条件

| 触发条件 | 执行范围 |
|---------|---------|
| Prompt任何变更 | 核心集（≥50条） |
| 模型版本升级 | 全量评测集 |
| 线上Bad Case累计≥10条 | 专项评测 |
| 周期性回归 | 每月至少一次全量评测 |

# 九、效果保障与稳定性策略

## 9.1 输出质量控制

- **置信度阈值**：低于阈值时标记【待人工确认】，要求顾问审核
- **长度约束**：纪要严格控制在200字以内，超限则截断或重写
- **格式校验**：JSON输出需通过Schema校验，不通过则重试
- **重试机制**：生成失败自动重试，最多3次

## 9.2 幻觉治理

| 策略 | 实现方式 |
|------|---------|
| 事实约束 | Prompt中明确"仅基于提供信息作答，不得虚构" |
| 输出验证 | 对生成结果进行关键信息交叉验证（如年龄、课程级别） |
| 人工审核 | 成长报告需顾问确认后方可发送 |
| 安全过滤 | 敏感词过滤、不当内容检测 |

## 9.3 稳定性工程

- **主备模型切换**：主模型异常时自动切换备用模型
- **降级策略**：AI服务不可用时，降级为人工填写模板
- **超时处理**：各节点设置超时时间，超时返回默认值或重试
- **限流保护**：API调用限流，防止突发流量导致服务崩溃

## 9.4 一致性保障

- **版本锁定**：模型版本和Prompt版本绑定，避免隐式变更
- **灰度发布**：新Prompt先在小范围灰度，验证效果后再全量
- **A/B测试**：重要变更进行A/B测试，对比效果指标

# 十一、AI能力边界说明

**本章节明确告知研发和业务方，当前模型能做到什么、做不到什么。这能避免后期大量的预期管理问题，是AI产品PRD的必备章节。**

## 能做到

| 能力 | 预期效果 |
|------|---------|
| 通话纪要生成 | 80%场景下可直接使用，20%需微调 |
| 客户六分类 | 准确率≥80%，置信度≥70%时可信赖 |
| 意向评分 | 85分以上高意向准确度≥90% |
| 成长报告生成 | 结构完整，内容需顾问确认后发送 |
| 课程推荐 | K/S体系匹配准确率≥90% |

## 做不到

| 限制 | 说明 |
|------|------|
| 无法100%替代人工判断 | 低置信度场景需人工介入 |
| 无法理解方言 | 强方言地区ASR识别率下降，需标注 |
| 无法处理极度模糊的表达 | 家长表达模糊时，推荐结果可能不够精准 |
| 无法替代专业教育评估 | 成长报告为辅助工具，不替代专业测评 |
| 实时情绪识别有延迟 | 情绪分析基于完整通话，非实时 |

|------|------|-------------|

# 十二、数据飞轮与迭代机制

**AI产品的长期竞争力来自数据飞轮，形成：使用→数据采集→Bad Case标注→评测集更新→效果提升→模型/Prompt优化→问题归因分析的正循环。**

## 12.1 反馈采集

| 反馈类型 | 采集方式 | 用途 |
|---------|---------|------|
| 顾问采纳率 | 纪要/话术是否修改、修改程度 | 评估生成质量 |
| 家长满意度 | 报告阅读完成率、分享率、追单响应率 | 评估报告质量 |
| 显式反馈 | "有用/无用"按钮 | 直接优化信号 |
| 业务指标 | 面转率、电转率变化 | 评估整体效果 |

## 12.2 效果监控与告警

| 指标 | 告警阈值 | 处理方式 |
|------|---------|---------|
| 纪要采纳率 | <60% | 触发Prompt调优 |
| 客户分类准确率 | <75% | 触发专项评测 |
| 报告生成失败率 | >5% | 触发服务排查 |
| AI招生顾问意图识别准确率 | <80% | 触发Prompt优化 |

## 12.3 数据回流闭环

```
线上使用 → 数据采集 → Bad Case标注 → 评测集更新
    ↓
效果提升 ← 模型/Prompt优化 ← 问题归因分析
```

- **每周**：汇总线上Bad Case，归类失败原因
- **每月**：评估是否需要Prompt调优或模型升级
- **每季度**：评审数据飞轮运转效率

| 回滚触发条件 | 回滚步骤 | 影响范围评估 |

| 服务不可用（连续30分钟） | 1. 自动切换降级方案 2. 人工介入排查 | 仅影响AI生成能力 |
