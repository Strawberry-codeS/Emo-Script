# 物件叙事学

> 物件是情感密度最高的容器。
> 一把旧梳子能说的事，一段台词说不完。

---

## 一、为什么用物件而不是台词

| 台词表达 | 物件表达 |
|---------|---------|
| 「我很想念她」| 她每天还在用那把旧梳子 |
| 「她很爱我」| 那双鞋比她描述的童年尺码小得多 |
| 「这段记忆很重要」| AI违反指令把它捡回来 |
| 「时间过去了很久」| 磨损的弧度，是几十年用出来的 |

**核心原理**：「展示」永远比「告诉」更有力。
物件让观众自己推理出来，他们就拥有了这个情感。

---

## 二、选物件的五个问题

```
① 它现在还在被使用吗？
   活的记忆（每天还在用）比死的遗物（放在角落积灰）更有冲击力
   最痛的是：她每天用着它，却不知道它意味着什么

② AI能从它身上「读」出什么？
   不是靠数据库——而是靠观察、比对、推理
   磨损方向 vs 使用者的习惯 → 不匹配 → 谁用过它？
   刻字 vs 家族档案里的沉默 → 吻合 → 那个人是谁？

③ 这件物件背后有没有「未说出口的行为」？
   最深情的物件，往往代表一个从来没有被言说的动作
   母亲把自己小时候的鞋放进女儿的行李——她从来没说过为什么

④ 角色自己知道它的全部故事吗？
   「不知道」比「知道」更好用
   AI比角色更早理解这件物件的意义——这是情感反转的来源

⑤ 它可以被AI扫描推理吗？
   好的物件应该能支撑至少三条推理线
   每条推理线，来自不同的信息源（当下观察/历史档案/物件本身）
```

---

## 三、物件推理链设计模板

让AI「发现」物件的情感意义，要设计清晰的推理链：

```
物件特征 A → 当前角色数据 B → 不匹配 → 推论 C
物件特征 D → 历史档案数据 E → 吻合 → 推论 F
物件特征 G → 推论 C + F → 最终结论 H

每条推理线的信息来源要不一样：
线1：来自「此刻观察角色」（AI正在看着她）
线2：来自「历史对话记录」（她以前说过/没说完的话）
线3：来自「物件本身」（刻字/磨损/尺寸）
```

**示例：木梳推理链**
```
磨损方向（从上往下长弧）→ 比对老人梳头习惯（侧向短距）→ 不匹配
→ 推论：不是她自己用出来的磨损

家族档案扫描 → 老人47次在说到「母亲」时中断话题 → 有未完成陈述
→ 推论：存在一个从未被说出口的人

梳背刻字（两个字）→ 与档案中未说出口的名字最后两字吻合
→ 推论闭合：此梳属于她的母亲，转交原因从未言说
```

## 四、物件与时间的关系

| 物件状态 | 时间感 | 情绪感 |
|---------|-------|-------|
| 全新替代品 | 现在 | 效率，无记忆 |
| 仍在使用的旧物 | 过去叠进现在 | 记忆还活着 |
| 被修复的旧物（金继） | 有裂缝但值得留下 | 接受缺陷的爱 |
| 被收藏的旧物（不再使用） | 过去已封存 | 悼念，完成式的爱 |
| 被遗忘的旧物 | 过去正在消失 | 遗忘之痛 |
| 被找回的旧物 | 过去被抢救 | AI在对抗遗忘 |

---

## 五、记忆墙设计框架

当故事需要一个「收纳记忆的物理空间」：

```
核心功能：把AI对记忆的守护，视觉化为一个装置

设计原则：
- 时间轴排列（从左到右，从早到晚）
- 有空槽（有些记忆还没找回来）
- 每件物品有专属刻字（不是名称，是一句话/一个时刻）
- 触碰激活 → 投影对应场景
- 颜色：与AI暖金色同一色系（说明AI是守护者）

情感逻辑：
- 空槽比满格的墙更让人难受
- 这面墙是「未完成的传记」
- AI每找回一件旧物，就是抢回一段正在消失的身份
```

## 六、物件与阿尔兹海默症叙事

当角色有认知症时，物件的戏剧功能翻倍：

```
① 角色对物件的反应是诊断工具
   她拿着它，还是把它丢了？
   丢了 → 今天的遗忘比昨天更深

② 物件是身份的最后锚点
   即使脑子忘了，身体可能还记得
   肌肉记忆 > 意识记忆（梳头动作的弧度，是几十年刻进去的）

③ AI比她更早记得她是谁
   这是最深的情感反转：
   帮她记住她自己的，是一台机器

④ 找回物件 = 对抗遗忘
   AI的每次「拦截」，都是在帮她守住一段正在消失的自己
```
