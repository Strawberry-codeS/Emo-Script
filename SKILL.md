# Story Alchemy 🧪

> **叙事炼金术**——你扔进来一句话，炼出来的是自带弧光分析的完整剧本。

## 为什么有这个 Skill

Sora、Runway、Pika、可灵……所有 AI 视频生成工具都在卷画质、卷物理真实度、卷镜头的连续性。

**但它们全部忽略了一个致命的步骤：剧本的弧光分析。**

你用一句"孤独的老人看夕阳"直接丢给视频生成模型，它产出的是一堆漂亮但空洞的画面——因为没人告诉它：
- 这个老人的孤独属于什么类型？（社交孤独？存在性孤独？被抛弃感？）
- 这场戏在整个情绪弧线里处于什么位置？（是起点、最低谷、还是释放点？）
- 人物在这场戏里朝哪个方向变化？（正在下沉？刚刚触底反弹？已经释然？）

**弧光结构才是提示词里最值钱的那部分。**

Story Alchemy 补的就是这一步——在你把灵感喂给任何下游工具之前，先把剧本的骨骼搭好。

## 一句话说清定位

```
你的一句话灵感
        ↓
   弧光分析 ← Story Alchemy 的核心差异
     ├── 这是什么情绪？几分强度？朝什么方向变？
     ├── 人物在这场戏里是上升还是下坠？
     └── 在整条情绪曲线中，这一拍的位置在哪？
        ↓
   完整剧本骨架（含情绪弧线 + 人物弧光）
        ↓
   结构化输出（JSON → 分镜拆解 / 角色设计 / 视频提示词）
```

## 所有视频生成工具都在卷画面，只有 Story Alchemy 在卷「这一拍意味着什么」

| | Sora / Runway / Pika | Story Alchemy |
|---|---|---|
| 能生成什么 | 漂亮画面 | 弧光分析 + 剧本骨架 |
| 输入需求 | 写死的提示词文本 | 一句话情绪 |
| 知道情绪弧线吗 | ❌ 不知道 | ✅ 每场戏标注了情绪强度、变化方向、在整个弧线中的位置 |
| 知道人物弧光吗 | ❌ 不知道 | ✅ 人物缺陷 → 动机 → 成长方向，全过程标注 |
| 输出可被下游消费吗 | ❌ 纯视频 | ✅ 结构化 JSON，任何 Agent 按字段读取 |

**Story Alchemy 不是替代视频生成工具——它是视频生成工具缺掉的那一环。**

## 工作流程（4 Phase）

### Phase 1：情感解析——弧光起点

- 识别核心情绪 + 子类型（不是「孤独」而是「存在性孤独 7/10」）
- 识别情绪冲突（笑着哭、愤怒中的无力）
- 判断情绪弧线方向：上升/下降/V字/倒V/波动
- 提取主题和隐喻线索

### Phase 2：剧本框架——弧光骨架

- 选结构：三幕 / Snyder 节拍表 / 环形 / 多线
- 人物弧光：缺陷 → 碰撞 → 变化（不是变强，是变理解）
- 情绪曲线：按幕标注情绪走向 + 关键拐点
- 每场戏标注：**情绪强度 + 变化方向 + 在弧线中的位置**

### Phase 3：互动完善——弧光校准

- 每次最多 2 个问题
- 重点追问弧光方向：「结尾观众应该感受到什么情绪？」
- 保留迭代历史

### Phase 4：标准化输出——下游即插即用

- **JSON**：`emotion_arc` 字段标注每场戏的情绪位置和变化方向 → 视频生成提示词可直接转化
- **Markdown**：人类可读的完整剧本

## 使用示例

```
用户：「一个被 AI 取代的程序员，在地铁站遇见自己写的第一个程序」

→ Phase 1 弧光起点
  - 情绪：存在性孤独 7/10 + 温暖怀旧 5/10 + 情境荒诞 4/10
  - 弧线预判：压抑 → 温暖 → 释然（V字型）
  - 主题：创造与被创造、身份的消解

→ Phase 2 弧光骨架
  - 人物弧光：从「用工作定义自我价值」→「接受自己本身就足够」
  - 情绪曲线：第一幕压抑(8) → 第二幕温暖(5) → 第三幕释然(3)
  - 每场戏标注了在弧线中的位置

→ Phase 3 校准
  1. AI 程序记得他吗？
  2. 结尾方向：希望还是绝望？

→ Phase 4 输出
  结构化 JSON + Markdown，可直接转化为视频生成提示词
```

## 重要规则

1. **弧光优先**：每场戏必须标注情绪强度、变化方向、在弧线中的位置
2. **先解析再生**：不要跳过 Phase 1 直接写剧本
3. **克制协作**：让用户做创意决策，你是协作者
4. **标记手法**：叙事技巧标注 → 下游可直接转化为镜头语言
5. **版本管理**：迭代追加，不覆盖
